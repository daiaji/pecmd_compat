# 003 PART 到 REGI

文档范围：`PECMD帮助文档.txt` 第 2255-2638 行。

本块覆盖 `PART`、`PATH`、`PBAR`、`PCIP`、`PINT`、`PUTF`、`RADI`、`RAMD`、`RAND`、`READ`、`RECY`、`REGI`。

## PART

功能点：
- 创建、删除、修改、隐藏、列表分区。
- 支持 MBR 分区表操作。
- 支持 GPT 分区表操作。
- 默认安全模式只允许操作一般不用的 3、4 分区，且不能是扩展分区。
- 支持图形模式。
- 支持只列出 USB 盘。
- 支持报告模式显示或输出列表，并可隐藏无盘符分区卷标。
- 支持高级模式操作 1、2 分区。
- 支持超级模式操作扩展分区。
- 支持起始对齐与尾部对齐，可指定 `64K`、`128K`、`512K`、`1M` 等。
- 支持强制清除分区内有效信息。
- 支持 raw 模式，不自动调整起始和长度。
- 支持更新磁盘后把分区表刷新到 OS。
- 支持移动磁盘盘符校正。
- 支持硬刷新移动磁盘。
- 支持强力模式硬刷新导致磁盘号重排。
- 支持新建分区时不弹出“未格式化”窗口。
- 支持指定 CHS：每柱面磁头数、每磁头扇区数、扇区大小。
- 支持以 `hd0` 到 `hdN` 或省略 `hd` 前缀指定磁盘。
- 支持操作主分区物理条目 1-4。
- 支持扩展分区最后增加逻辑分区 `+L`。
- 支持删除扩展分区最后逻辑分区 `-L`。
- 支持操作逻辑分区物理号 5-N。
- 支持设置或取消激活标志。
- 支持设置分区类型 0-255。
- 分区类型 0 表示彻底隐藏，分区软件和 OS 均不可见。
- 支持磁盘、分区、类型、起始、长度使用十六进制。
- 支持指定起始位置，从磁盘字节 0 开始，内部按 512 字节倍数缩小对齐。
- 起始位置可用 `-` 表示上一个分区结尾。
- 逻辑分区起始位置可加 `+` 表示跳过长度。
- 支持长度带单位后缀 `T/G/M/K/S`，`K=1024` 字节，`S=512` 字节。
- 支持 `del` 删除分区，不清除分区内数据，可重新映射加载。
- 支持 `init` 初始化 GPT/MBR 分区表。
- 支持 `fix` 修复 GPT 分区表校验和标志。
- 支持 `update`、`xupdate`、`hupdate` 刷新分区表到 OS。
- 支持 `list disk` 列出所有磁盘或返回指定磁盘信息。
- 磁盘信息包含磁盘号、大小、柱面数、磁头数、扇区数、扇区大小、媒体类型、签名、总线类型、分区类型、移动/固定磁盘。
- 支持 `list cdrom`、`list floppy`。
- 支持 `-devid` 返回磁盘 DevID。
- 支持 `-devidx` 返回物理序列号。
- 支持 `list part` 列出全部分区或返回指定分区信息。
- MBR 分区信息包含分区号、类型、引导标志、起始偏移、大小、隐藏扇区数、结尾、盘符。
- GPT 分区信息包含分区号、类型 GUID、GPT 标志、起始偏移、大小、属性、分区 GUID。
- 支持 `-hextp` 以十六进制显示类型。
- 支持 `-phy` 以物理分区号解释。
- 支持 `-phy#` 输出物理号。
- 支持 `-fill` 在没有盘符时输出占位符 `*`。
- 支持 `list drv` 按盘符列出分区信息并附加磁盘号。
- 支持 `-raw list drv` 返回设备号、分区号、盘符类型、总线、盘符、媒体类型。
- 支持 `-drv list volume` 返回卷、设备号、分区、设备类型、分区类型、偏移、大小。
- 支持 `list parent` 查询父设备。
- 支持 `list dep` 列出依赖源文件名。
- GPT 默认分区类型为 Microsoft Basic Data，默认属性为 0。
- GPT 支持插入模式，分区号前 `+` 表示原有分区后移。
- GPT 支持压缩分区表空条目。
- GPT 支持只读、影子、隐藏、无盘符、计算机必须分区等属性位。
- 支持重写 MBR，默认 `nt6/win`，也可指定 `nt5`、`dos` 或文件。
- MBR 写入源可为普通文件、EXEDATA 资源，并支持最大读取长度限制。
- 支持重写 PBR，默认 `nt6/win`，也可指定 `nt5`、`dos` 或文件。
- PBR 支持 FAT/FAT32/ExFAT 系统变体。
- PBR 写入源可为普通文件、EXEDATA 资源，并支持最大读取长度限制。
- 支持交换分区条目，如 `-swap:id`。
- 支持从磁盘镜像或磁盘对象操作分区，`-img=[*偏移*长度*]文件名|磁盘`。

Lua 重构提示：
- 应拆为 `disk.partition.list/create/delete/update`、`disk.mbr.write`、`disk.pbr.write`、`disk.gpt.*`。
- 这是高风险命令，Lua API 应默认只读或要求显式危险确认，不应默默复刻旧的危险模式。

## PATH

功能点：
- 设置 PECMD 当前工作目录。
- 建立目录，支持多级目录。
- 删除目录，也可能删除同名文件。
- 参数为空时恢复默认当前目录 `%SystemRoot%\SYSTEM32`。
- 前导 `@` 表示设置当前目录。
- 前导 `#` 表示删除目录。
- 无前导符表示建立目录。
- 文档警告配置文件中使用 `PATH @` 可能引起错乱。

Lua 重构提示：
- 映射为 `process.chdir`、`fs.mkdir_p`、`fs.remove`。

## PBAR

功能点：
- 在 `_SUB` 窗口内创建进度条控件。
- 支持自动回收。
- 支持形状、初始进度、文本、文本颜色。
- 支持光滑模式。
- 可通过 `ENVI @进度条=进度[;[#颜色:]文本]` 设置进度和文本。
- 进度为 `-1` 时隐藏，大于 100 忽略。
- 可通过 `ENVI @进度条.Visible` 设置可见状态。
- 可通过 `ENVI @进度条.color` 设置文本颜色。
- 可通过 `ENVI @进度条.percent=[-]smooth` 切换光滑模式。

Lua 重构提示：
- 映射为 UI progress bar。

## PCIP

功能点：
- 设置固定 IP、子网掩码、默认网关、DNS。
- 设置动态 IP，可单独指定 DNS。
- 查询网卡 IP、子网掩码、网关、DNS、名称、MAC、连接名、DHCP、状态、媒体、描述、类型。
- 查询模式支持返回实际值。
- 支持多个实现模式：保守、NT5 快速、默认稍快。
- 支持强制刷新或不刷新。
- 支持 NT5.x DHCP 到固定 IP 的可靠切换模式。
- 支持作用于全部网卡，或默认只处理以太网和无线网。
- 支持缓存变量。
- IP、掩码、网关、DNS 参数省略表示不修改。
- `-` 表示自动获取。
- `0.0.0.0` 表示清零。
- 支持多 IP、多掩码、多 DNS，分隔符可为 `;`、`:`、`&`。
- 网卡可按序号、GUID、描述、MAC、连接名称定位。
- 支持启用、禁用、刷新网卡状态。

Lua 重构提示：
- 映射为 `net.adapter.list/query/configure/enable/disable/refresh`。
- 现有 `win-utils.net` 是候选能力层，但需确认写配置能力。

## PINT

功能点：
- 将 EXE 或 LNK pin 到开始菜单或任务栏。
- 支持从开始菜单或任务栏脱离。
- NT5.x 只支持开始菜单。
- 支持指定菜单 DLL 文件。
- 支持通过菜单名称触发右键菜单项。
- 支持通过菜单标识 `StartMenu`、`TaskBand`、`-StartMenu`、`-TaskBand`。
- 支持通过 SHELL32.DLL 字符串 ID 定位菜单项。
- 三种参数菜单名称、菜单标识、字符串 ID 只取其一。
- 参数全省略时默认 pin 到开始菜单。
- 可借右键菜单名称执行其它右键菜单项目，如打开文件。
- Win10 下存在 Explorer 临时副本、注册表配置、菜单字符串配置、延迟处理等兼容逻辑。

Lua 重构提示：
- 现代 Windows pin/unpin 行为受系统限制，建议作为兼容可选功能或废弃。

## PUTF

功能点：
- 以二进制方式修改文件数据。
- 支持普通覆盖写，不改变文件长度。
- 新文件可创建，指定远超文件尾起始位置会形成稀疏洞。
- `PUTF*` 可扩大文件。
- 后缀 `#` 表示数据为 PE 对象。
- 后缀 `.` 表示直接写入 ANSI 文本。
- 支持 `-16`，数据队列按十六进制解释。
- 支持 `-v` 写 PE 变量。
- 起始位置支持十六进制和 `T/G/M/K/S` 单位后缀。
- 支持长度限制。
- 数据队列支持重复因子，如 `*1M 0x00`。
- 写入源可为数据队列、PE 变量、内部资源 ID、内部资源目录。
- 资源可指定 `EXEDATA` 下编号或非数字资源名。
- 支持 `-dd` 高速二进制块传送模式。
- `-dd` 支持输入/输出块大小、块数、长度、输入跳过块、输入跳过字节。
- `-dd` 支持输入为文件名、`zero`、整数。
- `-dd` 支持 `-conv=notrunc` 不截短已有文件。
- `-dd` 支持 `-quick` 稀疏文件、`-qspace` 快速分配空间、`-va` 不初始化空间。
- 支持写逻辑盘、光驱、磁带、物理磁盘、`NUL` 哑文件。
- 支持文件句柄加速。
- 支持特殊设备路径和 ImDisk 路径。
- 文件名以 `&` 开头表示 PE 变量。

Lua 重构提示：
- 映射为 `fs.raw.write`、`fs.copy_blocks`、`buffer.write`、`resource.read`。

## RADI

功能点：
- 在 `_SUB` 窗口内创建单选框控件。
- 支持右对齐、居中、缩放及内部图片缩放。
- 支持自动回收。
- 支持标题、点击事件、状态、组 ID。
- 状态支持选中、未选中、灰色不可用、不可见。
- 同组单选框互斥，不同组互不影响。
- 可通过 `ENVI @单选框=标题` 设置标题。
- 可通过 `ENVI @单选框.Check` 设置勾选状态。
- 可通过 `ENVI @单选框.Enable` 设置可用状态。
- 可通过 `ENVI @单选框.Visible` 设置可见状态。
- 可通过 `ENVI @控件.*del` 删除控件。

Lua 重构提示：
- 映射为 UI radio button。

## RAMD

功能点：
- 设置 RamDisk 大小。
- 旧格式支持按可用内存百分比、最小值、最大值、剩余值设置。
- 新格式支持 Ramdriv 和 ImDisk 驱动。
- 支持指定磁盘大小、格式、盘符、卷标、NTFS 压缩标志、物理内存方式。
- 支持 FAT、FAT32、NTFS 格式化。
- 使用 Ramdriv 时由驱动分配盘符和格式化。
- 使用 ImDisk 时可由 PECMD 分配盘符和格式化。
- 支持扩大指定 ImDisk 虚拟盘。
- 支持 ImDisk 完整命令格式 `-a` 创建、`-e` 编辑、`-d/-D` 卸载、`-l` 列表、`-gui` 图形模式。
- 支持 ImDisk 源文件/设备、大小、偏移、分区号、扇区大小、ImDisk 编号、几何参数、format 参数。
- 支持只读/可写、关机自动卸载、禁止关机自动卸载、关机全卸载、多分区可写控制。
- 支持通知盘符添加/删除。
- 支持不广播盘符。
- 支持扇区映射。
- 成功后设置进程环境变量 `%CurRamDisk%`。
- 内置 ImDisk 需要 `cabinet.DLL`。

Lua 重构提示：
- 映射为 `disk.ramdisk` / `imdisk` 兼容层。
- 需决定是否继续绑定 ImDisk 旧命令格式。

## RAND

功能点：
- 返回随机数。
- 默认写入 `RANDOM` 变量。
- 也可直接读取 `%RANDOM%`，不必显式调用。

Lua 重构提示：
- 映射为 `math.random` 或加密随机数 API。

## READ

功能点：
- 按字符串/行读取文本文件。
- 支持文件编码：Unicode、Unicode BE、UTF-8、GBK、BIG5、ANSI、数字代码页。
- BOM 优先于参数编码。
- 支持 `*fix`，孤立回车也当换行。
- 支持读取指定行号。
- 行号小于 0 表示读取行数。
- 行号 0 表示最后一行。
- 支持从指定行读取多行。
- 支持读取全部内容。
- 支持以换行或 CRLF 作为输出间隔，并可转换。
- 支持原始内容换行。
- 支持从多行字符串内容读取，而非从文件读取。
- 多行字符串可由 `EXEC*` 捕获或 `ENVI` 拼装。
- 支持从变量名读取被读内容，并可检测编码。
- 支持从标准输入读取，每次一行。
- 默认变量名为 `READLINE`。
- 支持 UD 文件。
- 支持特殊设备路径和 ImDisk 路径。

Lua 重构提示：
- 映射为 `text.read_lines`、`encoding.decode`、`stdin.read_line`。

## RECY

功能点：
- 设置 Windows 回收站。
- 支持 XP/2003/Vista/Windows 7 及其 WinPE。
- 可按指定分区设置。
- 可用 `*:\` 或 `*:` 表示所有分区。
- 最大容量在 NT5.x 为百分比，在 NT6.x 为 MB。
- 容量为 0 表示禁用回收站，删除文件不进入回收站。
- NT5.x 容量超过 30 时使用默认值 10。
- WinPE 中建议在 `INIT` 后使用。

Lua 重构提示：
- 映射为回收站策略/注册表配置。

## REGI

功能点：
- 读取注册表数据。
- 设置注册表数据。
- 删除注册表值。
- 删除整个注册表子项。
- 创建空项。
- 清空子项下默认值、子项、值。
- 列出全部值或全部子键。
- 支持快速模式。
- 支持提权。
- 支持读取失败时初始化为空。
- 支持数据变量名 `--name`。
- 支持只创建 key。
- 支持字节流模式。
- 支持不保存模式。
- 支持根键 `HKLM`、`HKCU`、`HKCR`、`HKU`、`HKCC`。
- 也可直接使用完整根键名如 `HKEY_LOCAL_MACHINE`。
- 支持默认键值。
- 支持查询值类型并返回类型变量。
- 支持任意类型读取。
- 支持查询未设置的空白值名 `NI`。
- 前导符 `$` 表示 REG_SZ。
- 前导符 `#` 表示 REG_DWORD。
- 前导符 `@` 表示 REG_BINARY。
- 前导符 `*` 表示 REG_MULTI_SZ。
- 前导符 `**` 表示 REG_EXPAND_SZ。
- 前导符 `*$` 表示多行串和 REG_MULTI_SZ 互转。
- 前导符 `~` 表示 REG_EXPAND_SZ。
- 前导符 `~~` 表示读取后再展开环境变量。
- 前导符 `^`、`+`、`b`、`u`、`n` 表示 REG_LINK、REG_QWORD、REG_QWORD_BIG_ENDIAN、REG_MUI_SZ、REG_NONE。
- 支持 `--t:数字类型` 直接指定任意类型，数据按 REG_BINARY。
- 支持 `--16` 把 binary/dword 数据按十六进制解释。
- 支持 `--0` 配合 `=` 建空项。
- 支持 `--0` 配合 `\!` 清空子项、值和默认值。
- 支持 `--0:N` 位标志，1 清默认值、2 删所有子项、4 删所有值，可组合。
- 支持空字符串、删除值、纯空白、前置/后置空白、写入 `""` 本身等细节语义。
- 支持 `TEAM` 包裹方式保留尾部空格。

Lua 重构提示：
- 现有 `win-kit.registry` 和 `win.reg` 可覆盖部分能力。
- 兼容层必须特别处理 REG_MULTI_SZ、REG_EXPAND_SZ、空字符串、删除值、删除 key、类型查询这几类易错语义。
